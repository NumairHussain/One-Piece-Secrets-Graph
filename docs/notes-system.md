# Notes System

## Purpose

The Notes System allows users to transform the graph from a visualization tool into a research environment.

Notes are entirely user-controlled.

They exist separately from canonical data.

---

# Note Types

## Node Notes

Attached directly to graph nodes.

Examples:

Joy Boy

Notes:

* Theory references
* Questions
* Important chapters

---

## Relationship Notes

Attached directly to graph relationships.

Example:

Robin → Joy Boy

Notes:

* Evidence
* Chapter references
* Interpretations

---

## Freeform Notes

Standalone notes not attached to graph entities.

Examples:

* Research journals
* Investigation boards
* Reading notes

---

# Markdown Support

Notes support Markdown.

Supported:

* Headers
* Lists
* Tables
* Blockquotes
* Code blocks
* Internal references

---

# Backlinks

Notes may reference:

* Characters
* Locations
* Events
* Secrets
* Other notes

Example:

[[Joy Boy]]

[[Void Century]]

[[Robin]]

---

# Embedded References

Users can embed graph entities inside notes.

Example:

Research note discussing:

* Joy Boy
* Ancient Kingdom
* Ohara

Referenced entities become clickable.

---

# Visibility Controls

Users may:

* Show all notes
* Hide all notes
* Show only workspace notes
* Show only node notes
* Show only theory notes
